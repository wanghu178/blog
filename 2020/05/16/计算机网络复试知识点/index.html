链接：https://www.nowcoder.com/discuss/1937

概述

1.协议模型

  1. OSI：物理层，数据链路层，网络层，传输层，会话层，表示层，应用层
  2. TCP/IP分层：网络接口层，网际层，运输层，应用层
  3. 五层协议：物理层，数据链路层，网络层，运输层，应用层

2.各层的协议及使用的设备

  1. 物理层：RJ45,IEEEE802.3（中继器，集线器，网关）
  2. 数据链路层：PPP, HDLC, WLAN,MAC，循环冗余校验，CSMA/CD（网桥，交换机）
  3. 网络层：IP,ICMP.ARP,RARP,OSPF,IPX,RIP,IGRP(路由器)
  4. 传输层：TCP,UDP,SPX
  5. 应用层：FTP,DNS,TeLent,SMTP,HTTP,WWW,NFS

3.各层的作用

  1. 物理层：通过媒介传输比特，确定机械及电气规范（比特BIT）
  2. 数据链路层：将比特组装成帧和点到点传输（帧frame）
  3. 网络层：负责数据包从源到宿的传递和网际互连（包Packet）
  4. 传输层：提供端到端的可靠报文传递和错误恢复（段Segment）
  5. 会话层：建立，管理和终止会话（会话协议数据单元SPDU）
  6. 表示层：对数据进行翻译，加密和压缩（表示协议数据单元PPDU）
  7. 应用层：允许访问OSI环境的手段（应用协议数据单元APDU）

协议

1. PPP

功能

  1. 在同一条物理链路上进行点对嗲的数据传输，对数据链路层的帧不进行纠错，不需要序号，不需要流量控制
  2. 封装成帧：加入帧界定符
  3. 透明性：字节填充法
  4. 多种网络层协议：在同一条物理链路上支持多种网络协议的运行
  5. 多种链路类型：PPP必须能够在多种类型的链路上运行（如串行和并行）
  6. 差错控制：接收方收到一个帧后进行CRC校验，若正确则收下，错误则丢弃
  7. 检测链接状态：自动检测链路是否处于正常工作状态

组成

  1. 提供一个将IP数据包封装到串行链路的方法。IP数据报在PPP帧中就时信息部分，长度收MTU限制
  2. 链路控制协议LCP：
  3. 一套网络控制协议NCP
  4. 认证协议：
  5. 注：支持异步链路（无奇偶校验和8比特数据）及面向比特的同步链路

2.HDLC

数据链路控制规程的功能

  1. 在物理层提供的通信接口的基础上，将易出错的数据电路构成相对无差错的数据链路
  2. 帧控制：发送站把网络送来的数据信息分为若干码组在每个码组加上地址段，控制字段，校验字段以及帧开始和结束标志，组装成帧来发送；要求接收端从收到的帧中去掉标志字段，还原成原始数据信息后送到网络层
  3. 帧同步：在传输过程中必须实现帧同步，以保证对帧中各字段的正确识别
  4. 差错控制：当数据信息在物理链路层中传输出现错误时，数据链路控制规程要求接收端能检测出错误并予以恢复，通常采用自动请求重发ARQ和前向纠错两种；当检测出无法恢复差错时，应通知网络层做相应处理
  5. 流量控制：对链路上的信息流量进行调节
  6. 链路管理：数据链路的建立，维护和终止
  7. 透明传输：
  8. 寻址：多点链路中，帧必须能够到达正确的接收站

HDLC–主站从站复合站

  1. 主站：发送命令帧，接受响应帧，对整个来南路的控制系统初启，流程的控制，差错检测或恢复
  2. 从站：接受由主站发来的命令帧，向主站发送响应帧，配合主站参与差错恢复等链路控制
  3. 复合站：既能发送，又能接受命令帧和响应帧，并且负责整个链路的控制

3.MAC

MAC地址用来识别数据链路层中相邻的节点（IP地址描述的是起点和终点）

4.CSMA/CD协议

多点接入(总线型)，载波监听，碰撞检测
